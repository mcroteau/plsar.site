PLSA.R

PLSAR is an Open Source Java Server + Framework for high demand, low latency requirements. There are no static references, no file reads, access to static fields per request. Everything is either cached and or instantiated on the fly. PLSAR runs via one command so there are no .war files to deploy, no additional plugins to install it is a simple yet sophisticated alternative to container deployment environments.

We even perform a lightweight dependency injection routine so you are not stuck boiler-plating.

We don't have official stats but it is estimated that PLSAR can handle 190,0000 concurrent connections per second.

We followed Spring so everything should be familiar.

Getting Started

Creating a simple REST endpoint is easy in non-persistent mode with no security. We can simplify the run & build process by installing the Gradle wrapper. This assumes you have Gradle installed.

We first need our build.gradle config file.

Dependency

implementation "net.plsar:plsar:1.05"

Main.java

public class Main {
    public static void main(String[] args){
        PLSAR plsar = new PLSAR(9001);
        plsar.start();
    }
}

HelloController.java

import net.plsar.annotations.Controller;
import net.plsar.annotations.http.Get;
import net.plsar.annotations.Text;

@Controller
public class HelloController {

    @Text
    @Get("/")
    public String index(){
        return "hi";
    }

}

PLSA.R runs via one command:

gradle run

Browse:

localhost:9001
